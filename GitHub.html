●GitHub（ギットハブ）で出来ること
  １）GitHub内に登録されたものは基本すべて公開(Public)される。限定公開（Private）も可能。
  ２）wiki（メモ書き）やタスク管理ツールなど、コラボレーションのための機能もある
  ３）GitHubを使うことで、リビジョン管理およびソースコード管理が可能
      具体的には下記です。
      ①リビジョン管理：前の修正内容を確認したり、ある時点の内容に戻すことが可能
      ②ソースコード管理：開発の課題や、ソースコードのチェック結果を共有することが可能
  ４）GitHub では2021年5月から .mp4 と .mov 形式の動画を 
      issue や pull request や discussion など に含められる。
      参考記事：https://www.itmedia.co.jp/news/articles/2105/18/news086.html
  ５）GitHubには、バージョン管理機能だけでなく、「開発を円滑に進めるための機能」がある。
      具体的には下記の3つです。
       １）フォーク：GitHub上の環境を自分のPCに入れる機能
       ２）プルリクエスト：自分の変更を他者に通知する機能
       ３）マージ：分岐した履歴を合体する機能


＜フォーク＞
具体的な内容は以下となります。
１）共有（きょうゆう）・・・他者が登録したコードをほかの人がコピーすることが「フォークを使う」という意味である。
２）試行錯誤（しこうさくご）・・・もとのコードを変えることなく、自分のところで改変が可能である。
３）流用（りゅうよう）・・・他者の資産をベースに自分本位のサービスを作ることができる。
※前述に示したとおり、他人のプロジェクトを自分のアイデアの出発点として使用するために使用されるのがフォーク機能である。


＜プルリクエスト＞
プルリクエストとは、自分のコードの変更を他者に通知し、変更内容をチェックしてもらう際に利用する機能である。
１）自分の変更作業がひととおり完了したことを他者に知ってもらえるから
２）ひとりで作ると気がつかないコードのバグや記述ミスを発見できるから
３）もしバグが発見されないまま登録されると、それを元に変更を加える人も困ってしまうから
※前述に示したとおり、自分の変更をあらかじめ他者にチェックしてもらうことで、登録されたコードの品質を保つために
  使われるのがプルリクエスト機能である。


＜マージ＞
マージとは、分岐した環境を統合する手段として利用される機能である。
具体的な内容は以下となります。
１）ブランチされた環境をひとつの環境(ブランチ元の環境)に戻すとき
２）プルリクエストの後に自分の変更を正式に登録するとき
※前述に示したとおり、マージ機能は、おもにブランチにより分岐した環境をブランチ元の環境に戻す際に使用される。
とくに、複数人での開発においては、人数分のブランチを切ることが一般的であることからマージ作業が発生するのが常でもある。

＜GitHubサインイン＞

＜新規WEBサイトを新規登録する方法＞
１）GitHubにサインインしたら、まずリポジトリを新規作成する
ルートリポジトリ「https://github.com/RobotWork01?tab=repositories」を選択し
右上の「New」ボタンをクリックすると「Create a new repository」が表示されます。
Owner「RobotWork01」であることを確認し、「Repository name」に新規WEBサイト名を入力する。
それ以外はデフォルトのまま変更しない。
「Public」を選択し、下の「Create repository」をクリックします。

２）リポジトリを新規作成したら、次にローカルPCからファイルをアップロードする。
（１）リポジトリページに移動したら、「<>code」タブを選択した状態で、
「Add file」ボタンを選択し、ドロップダウンメニューの中から「Upload files」を選択します。
「https://github.com/RobotWork01/RobotWork01/upload/main」が表示されたら
「Drag files here to upload」を表示させた状態で、予め準備しておいた 
 HTML ファイルをドラッグ＆ドロップしたあと、「Commit changes」ボタンをクリックする。

（２）次にリポジトリページで、「settings」タブを選択した状態で、
 まず、左側のサブメニュー「Pages」を選択した状態で、
「Branch」で「None」を選択してから「save」ボタンをクリックした後に
「Branch」で「main/root」を選択してから「save」ボタンをクリックする。
 次に「Build and deployment」のsourceで「GitHub actions」を選択して
 しばらく待つと公開されたURLが下記とおりに表示される。
 GitHub Pages is designed to host your personal, organization, or project pages from a GitHub repository.
 Your site is live at https://robotwork01.github.io/新しく作成したリポジトリ名/
 Last deployed by @RobotWork01 RobotWork01 19 hours ago

●GitHub LFS（Git Large File Storage、ギット・ラージ・ファイル・ストレージ）とは？

１）大容量ファイルが存在するパスに移動する。
２）git lfs 初期化する。
   ＜git lfs install＞
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git lfs install
   Updated Git hooks.
   Git LFS initialized.
３）git lfs track トラッキングする大容量ファイルを登録する。
   ＜git lfs track "*.jpg"、git lfs track "*.mp4"＞
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git lfs track "*.jpg"
   Tracking "*.jpg"
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git lfs track "*.mp4"
   "*.mp4" already supported

４）git add .gitattributes
   ＜git add .gitattributes＞
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git add .gitattributes
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>type .gitattributes
   # Auto detect text files and perform LF normalization
   * text=auto
   *.mp4 filter=lfs diff=lfs merge=lfs -text
   *.jpg filter=lfs diff=lfs merge=lfs -text
