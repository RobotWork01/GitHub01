●GitHub 開発手順




●GitHub LFS（Git Large File Storage、ギット・ラージ・ファイル・ストレージ）とは？

１）大容量ファイルが存在するパスに移動する。
２）git lfs 初期化する。
   ＜git lfs install＞
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git lfs install
   Updated Git hooks.
   Git LFS initialized.
３）git lfs track トラッキングする大容量ファイルを登録する。
   ＜git lfs track "*.jpg"、git lfs track "*.mp4"＞
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git lfs track "*.jpg"
   Tracking "*.jpg"
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git lfs track "*.mp4"
   "*.mp4" already supported

４）git add .gitattributes
   ＜git add .gitattributes＞
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git add .gitattributes
   C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>type .gitattributes
   # Auto detect text files and perform LF normalization
   * text=auto
   *.mp4 filter=lfs diff=lfs merge=lfs -text
   *.jpg filter=lfs diff=lfs merge=lfs -text

５）git commit -m "Add design file" コミットする。
  ＜git commit -m "Add design file"＞
  C:\Users\xxxxx\OneDrive\ドキュメント\GitHub\UiPath>git commit -m "Add design file"
  [main 7a5b2dc] Add design file
   1 file changed, 1 insertion(+)
